What is SYN-G-GEN
This is a visual programming tool for many kind of programming languages. You can create a state-chart and write program simultaneously.
Simple Instraction in chart
1. Create a state : Click on a space area (not on a state), then you will see a menu and select "New".
2. Move a state: Click and hold a mouse button on a state then move it.
3. Input program: Click on a state then select "Full edit", then you will see data table and input program.
4. Check program: Select a state then you will see converted source code in the left panel.
5. Connect states: Drag a red point on a state then drop it on a green point on a state.
6. Create branches: Select branch on state menu.
7. Convert : Push the convert button on the bottom left of window.
Template
A text for converting from data-table to source code.
There are two templates; source-template and function-template.
Note that source template is used in pre release version only.

Function template

[[itemname]]
Insert the value of the item name in the data table.
[[itemname->@macroname]]
Execute a macro using arguments of each line of the item's value.
[[itemname:line-number]]
Insert the line number value of the item's value.
[[::state:itemname]]
Insert the item value of the specified state. It allows you to use macro and line number attribute.
<<<?item-name
code part
>>>
If the value of the item name exists, the code part is validated.
<<<?item-name/regular-expression/
code part
>>>
Filter the value of item name with regular expression, and validate code part if the value exists.
<<<?"{%n}"/regular-expression/
code part
>>>
Filter the value of the argument 'n' with a regular expression, and validate the code part if the value exists.
<<<?xxxx
code-part
eof>>>
"eof>>>" will terminate converting.

MACRO

split string to arguments
Comma Separated Data: Arguments 0, 1,.
Example) tokyo, osaka, yokohama >> Argument 0 = "tokyo" Argument 1 = "osaka" Argument 2 = "yokohama"
Function: function name is assumed as argument 0 then the arguments of the function are argument 1, argument 2 · · ·.
Example) hoge (tokyo, osaka) >> Argument 0 = "hoge" Argument 1 = "tokyo" Argument 2 = "yokohama"
where to define macro
There are two places to define the macro
[macro] category in setting.ine and user macro file.
how to define macro
Macro name = converted value
Or
Macro name = @@@
Conversion value
@@@
how to use arguments
{%Number}
ie）
#Macro
hoge=I will go to {%0} and {%1}.
#Call
$hoge:tokyo,osaka$
#Convert
I will go to tokyo and osaka.
how to use arguments (delete double-quotes)
{%~Number}
Delete head and tail double-quotes of the argument string.
ie）

#Call
$hoge:"tokyo and osaka"$
#Convert
I will go to tokyo and osaka.

OUTPUT

Setting output
Output the conversion result in betwenn the output start and end marks in the source file.
ie)
// [SYN-G-GEN OUTPUT START] indent(4) $/./$
OUTPUT HERE !!
// [SYN-G-GEN OUTPUT END]
Output start mark
The output start mark is set at code_output_start of setting.ini.
After the start mark, set numbers of indent and conversion command.
Format: START-MARK indent (number) $conversion command$
Example)
[SYN-G-GEN OUTPUT START] indent(4) $/^S_/$
Indent 4 characters. Convert states starting with S_.
Converison command
$/Regular expression/$
After filtering each state with regular expression, convert states to program.
$/Regular expression/->#macro$
After filtering each state with regular expression, convert states to program using #macro.
※ The order of conversion is alphabetical order.
Output end mark
The output end mark is set at code_output_end of setting.ini.
End of output.

TIPS

Link to another state-machine
You can specify another state machine shortcut (psgg file) for the data table "state-ref".
The state menu shows the link. When clicked it opens the specified state machine.
Specify a psgg file without path.
Note: If there are multiple state machines of the same name, it will not work properly.
Create new state
A new state name is created using the last selected state name.
Therefore, when you want to create a state name of a series, it is convenient to create a new state once the original state is selected.
Explanation:
When the tool creates new state's name, the tool uses the latest selected state's name. If the selected state's name has numbers of its end, the tool increments the numbers. If the state's name does not have numbers of its end, the tool adds a number '1'. If the same name state exist, the tool increments the number.
